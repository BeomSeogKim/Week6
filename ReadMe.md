## 기능 구현 List
1. 회원가입
    -
    - ID 중복 검증 검사
    - PW 및 PW Confirm 유효성 검사 (기능 구현 예정) 
2. 게시글 작성
    -
    - 게시글 전체 조회 
    - 게시글 상세 조회 (댓글도 함께 조회)
    - 게시글 작성
    - 게시글 수정 및 삭제 (작성자만 가능)
3. 댓글 작성
    - 
    - 댓글 작성
    - 댓글 수정 및 삭제 (작성자만 가능)
4. 조회수 count
    -
    - 상세 게시글 조회시 게시글 조회수 증가
5. 좋아요 기능 구현
    - 
    - 회원은 각 게시글 및 댓글에 대해서 한번씩만 좋아요가 가능함
    - 좋아요를 누르지 않은 게시글 및 댓글에 대해서는 좋아요 취소가 불가능함
    - 회원의 좋아요 내역을 알 수 있도록 설계함
6. 마이페이지 기능 구현
    -
    - 회원이 작성한 게시글 조회 가능
    - 회원이 작성한 댓글 조회 가능
    - 회원이 좋아요를 누른 게시글 조회 가능
    - 회원이 좋아요를 누른 댓글 조회 가능


## TroubleShooting 

- CORS 정책에 의한 데이터 전송 불가  
    * => 서버에서 CorsFilter를 사용하여 해결
- CORS 정책에 의한 http으로 요청시 헤더에 토큰이 담기지 않는 현상 발생 
    * => 서버에서 CorsFilter를 사용하여 해결 
- 게시물 및 댓글 요청 응답시 시간 파일이 있을 때 Front 단에서 받기 힘들다는 피드백
    * => 게시물 및 댓글 관련하여 Response 내용 수정 
- 헤더에 Refresh-Token 으로 보낼 때 Front 단에서 "-" 를 처리하기 어렵다는 피드백 
    * => 헤더에 담기는 토큰 이름을 RefreshToken 으로 수정 
- 게시글 작성시 이미지업로드 기능 및 제목, 내용 따로 업로드하도록 설정했을 때 이 부분보다는 합쳐서 보내는 것이 나을 것 같다는 피드백 
    * => 사진은 Multipart 형식으로, 제목 및 내용은 json 형식으로 해서 이 전체를 form-data로 묵어서 처리할 수 있도록 서비스 변경 
- 댓글 요청 Request 관련하여 없어도 될 것 같다는 request 내용 관련한 피드백 
    * => 필요 없을 것 같은 request 목록 삭제
- 좋아요 기능 구현 관련하여 좋아요를 누르지 않은 회원이 좋아요 취소가 되는 현상을 발견 
    * => 회원의 활동내역을 추적하도록 post_like Entity 설정. 
    * => 좋아요를 누르지 않은 회원이 좋아요 취소를 누를 때 에러처리로 반영되지 않도록 변경
